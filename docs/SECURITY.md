# セキュリティポリシー

## 脆弱性の報告

**maimai.github.io** に脆弱性がある場合は、遠慮なく報告してください。

1. 以下のいずれかの方法で連絡してください：
   - Email: ytim8812@keio.jp
   - GitHub の [Security Advisory](https://github.com/yutojubako/maimai.github.io/security/advisories/new) を作成

2. 脆弱性を説明してください。

   修正案がある場合は、メッセージに添付または要約してください。

3. 私たちは脆弱性を評価し、必要に応じて修正またはそれに対処するための緩和策をリリースします。結果についてご連絡し、レポートにあなたをクレジットします。

   **修正がリリースされるまで、脆弱性を公開しないでください！**

4. 以下のいずれかの状況になった後、脆弱性を自由に公開できます：
   - a) 修正を公開した
   - b) 何らかの理由で脆弱性に対処しないことを決定した

## サポートされているバージョン

このプロジェクトは、最新のデプロイメントのみをサポートしています。GitHub Pages によって自動的にデプロイされる `master` ブランチが常に最新です。

## セキュリティのベストプラクティス

このプロジェクトでは、以下のセキュリティベストプラクティスに従っています：

- 依存関係を定期的に更新
- GitHub の Dependabot による自動セキュリティアップデート
- 最小権限の原則に基づいたアクセス制御
- 機密情報をコードに含めない

## 既知の制限事項

このプロジェクトは静的サイトであり、サーバーサイドの処理は含まれていません。したがって、一般的な Web アプリケーションの脆弱性（SQL インジェクション、XSS など）の多くは適用されません。
